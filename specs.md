## Guidelines

·                Open source first - Use libraries, frameworks, and tools you can name and justify. Prefer composing existing building blocks over reinventing everything (we value this highly).

·                Pipelines & automation - Show repeatable flows: ingest → transform → validate → output. Scripts are fine; clear, runnable pipelines are better.

·                Production mindset - Structure, configuration, error handling, tests where they matter, sensible secrets handling, and a path to deploy or run reliably (even if you only demo locally).

·                AI used well - Agents, LLMs, or classical ML are welcome when they solve a real part of the problem and aren’t a black-box wrapper around nothing.

·                Speed + clarity - We value working software and clear decisions over polish with no substance.


## Deliverables

1.                  Public GitHub repo with README: how to run, architecture in a few paragraphs, limitations, and third-party data/API notes.

2.                  ≤ 3 minute demo video - Walk through the running system and the pipeline; show the “why” not only the UI.

3.                  One-page write-up - Problem, approach, trade-offs, what you’d do next with another week, attach any valuable prompts you used while coding with AI tools.

## Challenge
You may build anything that fits the criteria below. If you want a concrete direction aligned with how we think about product and engineering review the following:

“Corporate Finance Autopilot”

Pick a listed public company. Using publicly available information and respectful scraping/API use:

1.                  Capture brand & positioning - e.g. from the company’s site, investor materials, or other public sources

2.                  Enrich with financial / market context - e.g. filings, investor pages, or reputable public data sources.

3.                  Generate structured output - e.g. a corporate presentation, financial model, information memorandum, prospectus, or a monthly shareholder report.

4.                  Financial reasoning layer - Build a financial model based on historical information that can forecasts a Base, Upside and Downside case (define inputs, link formulas, and ability to sensitise key drivers / assumptions.

5.                  Agentic or multi-step AI - e.g. planner + tools (retrieval, calculators, chart generation) with observable steps (logs or traces), not one opaque prompt.

6.                  Ability to ingest data - any source of company data extracted and structured into a company database that can be linked into the outputs above

7.                  Ability to visualise data - present a company to an equity or debt capital provide in a way they would want to assess an investment or credit case

8.                  Advisory - suggest for any chosen company what their funding and strategic options are

9.                  And anything else to blow our socks off!

Important: This is a student hackathon. Outputs are not investment advice. Label uncertainty, cite sources, and avoid presenting guesses as facts.
